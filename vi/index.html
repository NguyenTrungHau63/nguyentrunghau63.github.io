1. Tạo S3 Bucket và Website Hosting

Đầu tiên, chúng ta sẽ tạo một S3 bucket để lưu trữ mã nguồn của website.

  1. Truy cập AWS Management Console và tìm kiếm S3.

  2. Chọn Create bucket.

  3. Trong giao diện Create bucket:

    • Bucket name: Nhập nextgen-web-app (thêm một vài số phía sau nếu tên đã tồn tại).
    • Object Ownership: Chọn ACLs disabled (recommended).

    • Block Public Access settings: Giữ nguyên mặc định.

  4. Chọn Create bucket.

Tải dữ liệu lên S3

  1. Tải mã nguồn từ GitHub về máy: https://github.com/NguyenTrungHau63/NextGen.git.
  2. Mở thư mục nextgen-client trong Visual Studio Code.
  3. Chạy các lệnh sau trong terminal để build dự án: 
    npm i
npm run build
  4. Trong giao diện S3, chọn bucket vừa tạo và nhấn Upload.
  5. Chọn các file và thư mục trong thư mục nextgen-client/dist để tải lên.

Cấu hình Static Website Hosting

  1. Trong giao diện S3 bucket, chọn tab Properties.
  2. Kéo xuống và chọn Edit trong mục Static website hosting.
  3. Chọn Enable, sau đó chọn Host a static website.
  4. Nhập index.html vào trường Index document.
  5. Lưu lại các thay đổi.

Cấu hình Public Access và Bucket Policy

  1. Trong tab Permissions, chọn Edit ở mục Block public access (bucket settings).
  2. Bỏ chọn Block all public access và lưu thay đổi, sau đó xác nhận.
  3. Tiếp tục tại tab Permissions, chọn Edit ở mục Bucket policy.
  4. Dán đoạn policy sau để cho phép truy cập công khai vào các đối tượng trong bucket: 
    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::nextgen-web-app2406/*"
    }
  ]
}
  5. Nhấn Save changes.

2. Tạo DynamoDB Tables

Chúng ta sẽ tạo các bảng DynamoDB cần thiết để lưu trữ dữ liệu cho hệ thống.

  1. Truy cập DynamoDB Console và chọn Create table.
  2. Tạo bảng nextgen-tasks:
    • Table name: NextGenTasks
    • Partition key: id (String)
    • Sort key: timestamp (String)
  3. Tạo các bảng tương tự như sau:
    • nextgen-products (Partition key: id - String)
    • nextgen-categories (Partition key: id - String)
    • nextgen-banners (Partition key: id - String)
  4. Tạo bảng nextgen-users: NextGen-Tasks: NextGen-Metrics:

3. Tạo Lambda Function

Lambda Function sẽ là nơi chứa logic xử lý backend cho ứng dụng.

  1. Truy cập Lambda Console và chọn Create function.

  2. Cấu hình function:

    • Function name: nextgen-automation-api
    • Runtime: Node.js 22.x
    • Architecture: x86_64

  3. Nhấn Create function.

  4. Trong tab Code, dán mã nguồn được cung cấp.

    import { DynamoDBClient } from "@aws-sdk/client-dynamodb";
import { DynamoDBDocumentClient, ScanCommand, PutCommand } from "@aws-sdk/lib-dynamodb";
import { SNSClient, PublishCommand } from "@aws-sdk/client-sns";

const dynamoClient = new DynamoDBClient({ region: "us-east-1" });
const dynamodb = DynamoDBDocumentClient.from(dynamoClient);
const sns = new SNSClient({ region: "us-east-1" });

export const handler = async (event) => {
    console.log('Event:', JSON.stringify(event, null, 2));

    const headers = {
        'Content-Type': 'application/json',
        'Access-Control-Allow-Origin': '*',
        'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
        'Access-Control-Allow-Headers': 'Content-Type'
    };

    try {
        const { httpMethod, path } = event;

        if (httpMethod === 'GET' && path === '/api') {
            return await getDashboardData(headers);
        }

        if (httpMethod === 'POST' && path === '/api') {
            const body = JSON.parse(event.body || '{}');
            return await triggerAutomation(body, headers);
        }

        return {
            statusCode: 404,
            headers,
            body: JSON.stringify({ error: 'Not Found' })
        };

    } catch (error) {
        console.error('Error:', error);

        return {
            statusCode: 500,
            headers,
            body: JSON.stringify({ error: error.message })
        };
    }
};

async function getDashboardData(headers) {
    try {
        const command = new ScanCommand({
            TableName: 'NextGenTasks'
        });

        const tasks = await dynamodb.send(command);

        return {
            statusCode: 200,
            headers,
            body: JSON.stringify({
                metrics: {
                    users: 150,
                    tasks: tasks.Items.length,
                    alerts: 3
                },
                status: 'operational',
                timestamp: new Date().toISOString()
            })
        };
    } catch (error) {
        throw new Error(`Dashboard error: ${error.message}`);
    }
}

async function triggerAutomation(data, headers) {
    try {
        const task = {
            id: Date.now().toString(),
            task: data.task || 'Default automation task',
            status: 'pending',
            timestamp: new Date().toISOString(),
            priority: data.priority || 'medium'
        };

        const putCommand = new PutCommand({
            TableName: 'NextGenTasks',
            Item: task
        });

        await dynamodb.send(putCommand);

        // Send SNS notification
        const publishCommand = new PublishCommand({
            TopicArn: 'arn:aws:sns:us-east-1:048244750537:nextgen-alerts',
            Message: `New automation task: ${task.task}`,
            Subject: 'NextGen Automation Triggered'
        });

        await sns.send(publishCommand);

        return {
            statusCode: 200,
            headers,
            body: JSON.stringify({
                message: 'Automation triggered successfully',
                taskId: task.id
            })
        };
    } catch (error) {
        throw new Error(`Automation error: ${error.message}`);
    }
}

  5. Nhấn Deploy để lưu lại code.

4. Tạo API Gateway

API Gateway sẽ đóng vai trò là điểm cuối (endpoint) cho ứng dụng frontend.

  1. Truy cập API Gateway Console và chọn Create API -> REST API.

  2. API name: nextgen-api

  3. Endpoint type: Regional

  4. Tạo một resource có tên api với path /api.

  5. Trong resource /api vừa tạo, thêm một method ANY.

  6. Tích hợp method này với Lambda Function nextgen-automation-api.

  7. Tương tự cho Method: GET.

  8. Sau khi cấu hình xong, chọn Actions -> Deploy API để triển khai.

  9. Trong AWS Console, truy cập API Gateway.

  10. Chọn API nextgen-api của bạn.

  11. Trong phần Resources, click chuột phải vào resource /api và chọn Create Resource.

  12. Điền các thông tin sau:

    • Resource Name: dashboard
    • Resource Path: /dashboard
    • Nhấn Create Resource.

  13. Tạo file Dashboard HTML

Tiếp theo, bạn sẽ tạo một file HTML đơn giản để hiển thị dữ liệu từ API.

  • Tạo một file mới với tên dashboard.html và dán đoạn code sau vào:
<!DOCTYPE html>
<html>
<head>
    <title>NextGen Dashboard</title>
    <style>
        body { font-family: Arial, sans-serif; margin: 20px; }
        .card { border: 1px solid #ddd; padding: 20px; margin: 10px; border-radius: 8px; }
        .metric { font-size: 2em; color: #007bff; }
    </style>
</head>
<body>
    <h1>NextGen Dashboard</h1>
    <div id="dashboard">Loading...</div>
    <script>
        function loadData() {
            fetch('[https://teq8fa3cy3.execute-api.us-east-1.amazonaws.com/prod/api](https://teq8fa3cy3.execute-api.us-east-1.amazonaws.com/prod/api)')
                .then(response => response.json())
                .then(data => {
                    document.getElementById('dashboard').innerHTML =
                        `<div class="card">Tasks: <span class="metric">${data.metrics.tasks}</span></div>
                         <div class="card">Users: <span class="metric">${data.metrics.users}</span></div>
                         <div class="card">Alerts: <span class="metric">${data.metrics.alerts}</span></div>`;
                })
                .catch(() => {
                    document.getElementById('dashboard').innerHTML = 'API Error';
                });
        }
        loadData();
        setInterval(loadData, 30000);
    </script>
</body>
</html>
  1. Lấy API URL để test: https://teq8fa3cy3.execute-api.us-east-1.amazonaws.com/prod/api

5. Cấu hình SNS Topic

SNS được sử dụng để gửi các thông báo tự động.

  1. Truy cập SNS Console và chọn Create topic.
  2. Name: nextgen-alerts
  3. Type: Standard
  4. Sau khi tạo topic, chọn Create subscription.
  5. Protocol: Email
  6. Endpoint: Nhập địa chỉ email của bạn.
  7. Mở email và xác nhận subscription.

6. Cấp quyền cho Lambda

Để Lambda Function có thể truy cập DynamoDB và SNS, bạn cần cấp quyền IAM cho nó.

  1. Truy cập Lambda Console, chọn function nextgen-automation-api.
  2. Vào tab Configuration -> Permissions.
  3. Click vào Role name để mở IAM Console.
  4. Thêm các chính sách (policies) sau vào role:
    • AmazonDynamoDBFullAccess
    • AmazonSNSFullAccess
  5. Lưu lại các thay đổi.

7. Thiết lập CloudWatch Dashboard và Alarms

CloudWatch giúp giám sát hệ thống và thiết lập cảnh báo khi có lỗi.

Tạo Dashboard

  1. Truy cập CloudWatch Console -> Dashboards -> Create dashboard.
  2. Dashboard name: NextGen-Monitoring
  3. Thêm các widgets để theo dõi các metric của Lambda và API Gateway, ví dụ: Invocations, Errors, Count (của API Gateway).

Tạo Alarm

  1. Trong CloudWatch, vào Alarms -> Create alarm.
  2. Chọn metric Errors của Lambda Function nextgen-automation-api.
  3. Đặt điều kiện: Whenever Errors is Greater/Equal (>=) than 3.
  4. Cấu hình action để gửi thông báo đến SNS topic nextgen-alerts khi alarm được kích hoạt.
  5. Đặt tên alarm là NextGen-Lambda-Errors và lưu lại.

Dọn dẹp tài nguyên

Xóa S3 Bucket

  1. Mở Amazon S3 Console.

  2. Chọn bucket nextgen-web-app (hoặc tên bucket bạn đã tạo).

  3. Nhấn Empty để xóa toàn bộ object.

  4. Xác nhận thao tác bằng cách nhập tên bucket, sau đó nhấn Empty.

  5. Sau khi bucket rỗng, chọn Delete bucket → nhập tên bucket → Delete.

⚠️ Lưu ý: Việc xóa bucket sẽ xóa toàn bộ dữ liệu lưu trữ bên trong.

Xóa DynamoDB Table

  1. Mở Amazon DynamoDB Console.

  2. Chọn bảng NextGenTasks.

  3. Nhấn Delete table.

  4. Xác nhận xóa bằng cách nhập tên bảng và nhấn Delete.

Xóa Lambda Function

  1. Mở AWS Lambda Console.

  2. Tìm và chọn function nextgen-automation-api.

  3. Từ menu Actions, chọn Delete.

  4. Xác nhận bằng cách nhấn Delete.

Xóa API Gateway

  1. Mở Amazon API Gateway Console.

  2. Chọn API nextgen-api.

  3. Chọn Actions → Delete.

  4. Xác nhận bằng cách nhấn Delete.

Xóa SNS Topic

  1. Mở Amazon SNS Console.

  2. Chọn topic nextgen-alerts.

  3. Nhấn Delete → Xác nhận.

Xóa CloudWatch Dashboard & Alarm

Xóa Dashboard

  1. Mở Amazon CloudWatch Console.

  2. Trong menu Dashboards, chọn NextGen-Monitoring.

  3. Nhấn Actions → Delete dashboard → Xác nhận.

Xóa Alarm

  1. Trong CloudWatch Console, chọn Alarms.

  2. Tìm alarm NextGen-Lambda-Errors.

  3. Chọn Actions → Delete → Xác nhận.

7. Gỡ IAM Role và Policy (nếu tạo riêng cho lab)

  1. Mở IAM Console.

  2. Vào Roles, tìm role đã gán cho Lambda hoặc API Gateway.

  3. Nhấn Delete role → Xác nhận.

  4. Nếu tạo riêng policy cho lab, vào Policies, chọn policy → Delete.

Hãy kiểm tra lại toàn bộ tài nguyên trong AWS Console sau khi xóa để đảm bảo không còn dịch vụ nào đang chạy gây phát sinh chi phí.
